Chapter 6: Programming Languages Terminology

Term Definition
Machine Language A computer programming language consisting of binary or hexadecimal instructions that a computer can understand and respond to directly.
Debugging The process of correcting errors in a computer program. Errors can be syntax, semantic, or run time.
Assembler A program that converts mnemonic expressions into a pattern of bits that the computer can understand and use to perform basic operations.
Assembly Language A mnemonic programming language system that communicates with the hardware of a computer directly. It uses code words and expressions that can be easier to understand and interpret than the binary or hexadecimal data the computer stores and reads.
Translator Translates programs expressed in high-level primitives to machine language programs. It converts high-level programming languages into patterns of bits that the computer can understand.
Compiler Translates high-level programming code into machine-code or lower-level forms that can be read and executed by a computer.
Interpreter Transforms or interprets high-level programming code into machine language that can be executed. An interpreter reads each statement of code and then converts or executes it directly.
Cross-platform Software Software that is independent of an operating system along with a machines hardware design and is executable throughout the network.
Natural Language A language that has developed naturally over time without formal grammatical analysis.
Formal Language A language designed for use in specific situations, such as mathematics or computer programming.
Machine Independence Software that is not dependent on the properties of a particular machine, and can be used on any computer system.
High-level Programming Language A high-level language is any programming language that enables development of a program in a user-friendly programming environment and is typically machine independent.
Object-oriented Programming (OOP) The software system is seen as a collection of units, called objects, that are capable of performing actions to itself or other objects.
Scripting Language A programming language that employs a high-level construct or interpreter to interpret and execute one command at a time.
Data Types:
  • Integer: Numeric data that is positive and negative whole numbers.
  • Float: Numeric data that is decimal value or fractional number.
  • String: A sequence of characters used to represent text.
Primitive Data Type A set of basic data types from which all other types are constructed.
Data Structure A specialized format for organizing, processing, retrieving and storing data. The conceptual shape or arrangement of data in variables.
Python Collection Data Structures
  • List: A mutable ordered sequence of elements.
  • Tuple: A immutable data structure that contains related elements.
  • Set: A mutable unordered set of elements that cannot include duplicates.
  • Dictionary: A collection of key and pair values where the key value is immutable and the value can be any type.
Function A set of instructions for performing a task that can be used as an abstract tool by other program units.
Local Variable A variable declared within a part of a program like a function and can only be used within that part of the program.
Global Variable A variable whose scope is not restricted to a particular part of a program and it can be used throughout the program.
Source Program A computer program written in a higher-level programming language that will be converted into an equivalent object program by a compiler or assembler.
Object Program A computer program translated from an equivalent source program into machine language by a compiler or assembler.
