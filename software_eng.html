Chapter 7: Software Engineering
Term Definition
Software Engineering Establish principes that guide the software development process and lead to the development of efficient and reliable products.
Integrated Development Environment (IDE) A developing environment that combines many tools for software development into a single integrated package. The tools can include editors, debuggers, and compilers or translators.
Association for Computing Machinery (ACM)
  • Purpose of ACM: Advancing the art, science, engineering and application of information technology; fostering the open interchange of information to serve both professionals and the public; and promoting the highest professional and ethical standards.
  • Benefits of ACM membership: includes online subscription, books, courses, videos, and tutorials on job skills and certifications. Technical talks, access to the ACM digital library, and discounts on registrations to ACM special interest conferences are also included with membership.
  • Student ACM membership: There are several student membership options that range in price from $19 to $62 USD. There is also a student transition membership.
Institute of Electrical and Electronics Engineers (IEEE)
  • Purpose of IEEE: The core purpose is to foster technological innovation and excellence for the benefit of humanity.
  • Benefits of IEEE membership: Network with other technological professionals and participate in discussions on technical topics. Establish a professional profile in your account abd creates groups to share and collaborate on projects. Discover IEEE events and activities throughout the world.
  • Student IEEE membership: A student membership is available, but the student must carry be at least 50% full time in a regular course of study in a IEEE designated field. The cumulative period for a student membership grade is eight years.
Software Life Cycle: Development Phase
  • Requirement Analysis: The process of determining the user needs and expectations for a new or modified software product.
  • Design: The process of defining software solutions to one or more sets of problems. It involves creating a plan for construction of the proposed system.
  • Implementation: The process and procedures for taking software applications and tools from planning and design to production. It involves writing programs, creating data files, and development of databases.
  • Testing: The process of evaluating and verifying that a software product or application works as intended. This can include checking for bugs, reducing development costs, and improving performance.
Modular Implementationr Development of software applications with manageable units, called modules, each of which deals with only a part of the overall software system.
Unified Modeling Language (UML) A general purpose modeling language that is used to provide a standard way to visualize the design of a software system. It is set of rules specifically for drawing diagrams that includes object oriented programming.
Use Case Interactions between the program system and it users. Used in use case diagram representations.
Use Case Diagram A diagram that shows the proposed system as a large rectange in which interactions between the system and its users are represented by ovals and users are by stick figures..
Data Relationships:
  • One to One: A data class relationship where there is one class and one association.
  • One to Many: A data class relationship where there is one class and many associations.
  • Many to Many: A class relationship where there are many classes and many associations.
Design Pattern A predeveloped model for solving a recurring problem in software design.
Software Quality Assurance (SQA) An ongoing process that ensures the software product meets and complies with the organization's established and standardized quality specifications.
Software Testing
  • Glass Box Testing: The software tester is aware of the interior structure of the software and uses this information to design the test.
  • Black Box Testing: Testing that does not rely on knowledge of software's interior composition. It is performed from the user's point of view only.
  • Beta Testing: A preliminary version of the software is given to users with the goal of learning how the software performs under real conditions before the final software is released.
  • Alpha Testing: Testing similar to beta testing, but the actual testing is performed at the developers site.
Documentation
  • User Documentation: Explains the features of a software application and how to use them.
  • System Documentation: Describes the software application's internal composition so the system can be maintained during its life cycle.
  • Technical Documentation: Describes how the software application should be installed and serviced.
Intellectual Property Law:
  • Software License: A legal agreement between the owner and user of a software product that grants certain permissions to use the product.
  • Software Disclaimer: A notice to the user that the developer is denying any and all liability for any damages arising out of using the software as well as denying any implied warranties.
  • Best defenses against software liability claims: Applying sound software engineering principles during development, using a care level compatible with the software's application, and producing while also maintaining records that validate this effort.
