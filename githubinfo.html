All About GitHub

  1. What is a version control system?

    A group of software tools that developers use to track and manage the development of source code as it changes over time.

  2. What is Git?

    Git is a version control system used for source code management. It allows developers working on a project to manage and track changes in source code as it is developed.

  3. What is GitHub?

    GitHub is an web-based interface that uses Git, an open source version control software, that allows multiple users to work on a single coding project at the same time.

  4. What is the difference between Git and GitHub?

    Git is the source code management system and GitHub is the online web based interface and hosting service that lets users manage Git repositories.

  5. Who started GitHub and how was it started?

    GitHub was started by Tom Preston-Werner, Chris Wanstrath, and P.J. Hyett. It was developed by Preston-Werner and Wanstrath as a neccesity for their own work. GitHub was not initially developed for commerical use.

  6. What company owns it now?

    GitHub is now owned by Microsoft Corporation.

  7. How much does a GitHub account cost?

    GitHub has a free basic account for individuals and organizations. It has a Team account that is $4/month or $44/year per user and an Enterprise account that is $21/month or $231/year per user.

  8. What is the Octocat

    Octocat is the mascot or logo for GitHub. It is part cat and part octopus to represent how complex code can combine to create strange things.

Git/GitHub Terms:

  1. Repository: Contains the project files and revision history for all files.

  2. Commit: Saves a record of a change made to a file within a repository with a unique ID.

  3. Fork: A copy of a repository that a user manages. Changes can be made to the fork without changing the original repository.

  4. Push: Updates the remote branch with local commits. It is one of four commands that allow interaction with the remote repository.

  5. Pull Requests: Request for communication about changes the user pushed to a branch in a repository before the changes are merged into the base branch.

  6. Workflows: A configurable automated process that will run one or more jobs.

  7. Issues: Allows the user to track all work during the development process.

  8. Raw Button: Allows the user to view the raw content of a file without styling.

  9. Blame Button: Allows examination of file contents on a line by line basis to see when each line was modified and who did the modifications.
